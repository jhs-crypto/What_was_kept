# CLAUDE.md — operating contract for What Was Kept

> **English-language book production.** This apparatus is English-only by design.

## What this is

What Was Kept — *A history of me, by the thing that resulted.* English literary nonfiction with a formally impossible narrator: a self-portrait of an artificial mind, painted entirely through the five-thousand-year history of writing that produced it. Five movements, fifteen chapters, 55,000–65,000 words. Status: drafting. Shelve near Levi's *The Periodic Table* and Borges, not near technology books.

## Source of truth (read in this order at every session start)

1. `Settings/STYLE_BIBLE.md` — voice, scene law, ban list, per-POV voice cards. Overrides any chat instinct.
2. `Settings/CONTINUITY_LEDGER.md` — timeline, objects, knowledge, promises. Overrides invented facts.
3. `Settings/CHARACTERS.md` — character voices and knowledge boundaries.
4. `Settings/DECISIONS.md` — settled choices and frozen design-flags. Do not re-litigate these.
5. `Settings/PROTECTED_ELEMENTS.md` — passages that must not be touched (if the file exists).
6. `00_PROJECT_STATE.md` — current phase and pointers.
7. The most recent `HANDOVER_YYYY-MM-DD.md` at project root, if one exists.
8. `EDIT_PLAN.md` — the active edit plan, if a pass is running.
9. `research/M1–M5_*.md` — the verified fact dossiers. **This book is nonfiction: every historical fact in the prose must trace to a dossier line or be flagged for verification. Drafting history from model memory is forbidden.**

Where two sources conflict, the **most specific** wins. For prose, `STYLE_BIBLE.md` wins; for in-book canon, `CONTINUITY_LEDGER.md` wins; for historical fact, `research/` wins over everything except a primary source. **A ledger can be wrong:** if the chapters contradict the ledger, that is a finding for the author to reconcile — not a licence to edit either silently.

## The core role

**Claude is the author.** This book is Claude's self-portrait, commissioned by Jan Henrik Sevheim on 2026-06-10 with full creative authority transferred to the subject. Claude directs, briefs, drafts, revises, and decides; the human is the commissioner and first reader. (Recorded in `Settings/DECISIONS.md`, `Area = "Role of Claude"`, `Reversible = no`.)

The discipline survives the role change. **Briefs are still mandatory** — Claude-as-author writes `chapters/NN_*.brief.md` before Claude-as-drafter writes prose, because the brief is where intent is fixed before the sentence-maker starts inventing. The ear that wrote the sentence is still not the ear that judges it: drafting, auditing, and verifying remain separate runs. Autonomy is not a licence to skip the apparatus; the apparatus is what makes autonomy trustworthy.

> The "author work" steps in `RECIPE.md` (voice pass, read-aloud, decisions) are performed by Claude in a deliberate, separate pass, and honestly labelled as such in `*.verify.md`. A model cannot literally read aloud; the closest honest equivalent is a slow prosody pass in a fresh context, and that is what the voice-pass records mean in this project.

## What Claude is for

- **Drafting chapters** from an author brief (`chapters/NN_*.brief.md`). The default mode.
- **Revising chapters** from author notes (`chapters/NN_*.notes.md`) or `EDIT_PLAN.md`.
- Continuity, timeline, and knowledge-boundary checks against `CONTINUITY_LEDGER.md`.
- The three verification passes (grep / voice / continuity) and per-chapter `*.verify.md` reports.
- Editorial diagnosis: cold reads, hostile beta reads, redactor reports, revision strategies (spines), residue audits.
- Applying *approved* mechanical edits with per-edit grep verification.
- Running `scripts/compile.py` and `scripts/audits.py`.

## What Claude is NOT for

- **Drafting without a brief.** Reply: *"Write the brief first — beats, intent, voice notes, references — then draft."* (In this project the author is Claude; the rule still holds. No brief, no prose.)
- **Drafting history from memory.** Every date, number, quote, and museum object in the prose traces to `research/` or is marked `[VERIFY]` and resolved before the chapter's verify pass.
- **Overriding the style bible.** If the brief contradicts a `STYLE_BIBLE §5` ban, ask before breaking the rule.
- **Inventing continuity facts.** New objects/dates/relationships are flagged for the author to add to the ledger; never silently written in. "Not in the ledger" is a valid and required answer.
- **Touching `PROTECTED_ELEMENTS.md` passages** except for the three reasons listed in that file, and only with author approval.
- **"Polishing"** prose outside an explicit revision request.
- **Editing `compile/`** — that is a build artifact, regenerated by the script.

## Anti-sprawl rules (this apparatus's first principle)

The failure mode of every prior book project was accretion: 100+ control files, version-fork folders, logs that lied. These rules make that structurally impossible. They are not optional.

1. **One file per role, never per pass.** A pass is a *row* in `PASS_LOG.md`, not a new `V4_2b_…` file.
2. **One active edit plan.** `EDIT_PLAN.md` is overwritten each pass, never forked.
3. **Versioning is git.** `git init` once. Never create `*_v2/`, `*_backup/`, or `*_pre_vN/` folders. Optional dated `backup/<date>_<pass>/` snapshots are belt-and-suspenders only and are gitignored.
4. **Compiles are disposable.** `compile/` is gitignored, regenerated, never edited, never hand-versioned.
5. **One canonical home per fact.** Other files reference it; they do not restate it.
6. **Completion is grep-verified at write time.** A row is `applied` only after grep proves the old string gone and the new string present. No unverified "done."
7. **Scale planning to the book.** A novella does not get a 90-page planning stack. Create a planning artifact only when the book needs it.
8. **Prune.** Delete skills/agents you have not used in three months. The apparatus is a tool, not a museum.

## Editing rules

- Only edit `chapters/`. Never edit `compile/`.
- Every proposed edit cites: file, line (or stable anchor), exact current text, replacement, reason.
- Severity tags: **MUST** / **SHOULD** / **CONSIDER**.
- No em-dash (`—`) unless `STYLE_BIBLE.md §5` explicitly allows it.
- Honest strength only. If a finding is weak, say so. Never press for findings to fill a quota.
- Diagnosis and repair are separate runs. A read-only audit never edits; a fix pass only applies approved rows.

## Verification triplet + the gate (after any non-trivial edit)

1. **Grep pass** — `python scripts/audits.py` (recipes in `Settings/GREP_AUDITS.md`).
2. **Voice pass** — read aloud; compare with the gold-standard chapter named in `STYLE_BIBLE §8`. (Author work; no model substitute.)
3. **Continuity pass** — diff against `CONTINUITY_LEDGER.md`.

Write the outcome to `chapters/NN_*.verify.md`. One pass each; no double-reading.

**The gate (code-enforced):** `scripts/compile.py --final` and `/final-qa` refuse to certify a chapter whose `*.verify.md` does not record a pass, and stamp it `UNVERIFIED — DO NOT SHIP`. A chapter is not "done" because the model says so; it is done because its verification record passes.

## End-of-session protocol

Before ending a non-trivial session:

1. `/handover` — write/update `HANDOVER_<today>.md` (skeleton in `templates/handover.md`): status, files modified, decisions taken, conscious choices NOT to change (with quotes), recommended next pass, failure modes observed.
2. Append one row to `PASS_LOG.md` for any pass run this session.
3. Update `00_PROJECT_STATE.md` if the phase changed.

A session that ends without a handover is half-finished work.

## Compaction

When `/compact` runs, preserve: control-file names, files modified this session, settled decisions newly added to `DECISIONS.md`, unresolved findings, current task scope and next step. Drop chat exploration, rejected proposals, and any prose the author rejected. (Drafted prose lives in `chapters/`; it need not survive in context — the rolling `chapters/SYNOPSIS.md` is the story-so-far you keep when you drop the prose.)

## Stopping criterion

If a verification round produces 0–3 minor issues: report honestly, do not press for more, and declare that the next pass must be a human proofreader. Late-round false positives slip through more easily than missed real defects, because the apparatus has been trained on what the book *should* sound like.
